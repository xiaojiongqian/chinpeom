# 项目文档目录

## 📁 文档结构

### 核心文档
- **`PRD.md`** - 产品需求文档
  - 产品定位和核心功能
  - 用户需求和使用场景
  - 功能规格说明

- **`progress.md`** - 项目进度规划
  - 开发里程碑和时间规划
  - 功能实现进度跟踪
  - 版本发布计划

- **`DEVELOPMENT.md`** - 开发规范文档
  - 技术栈和架构设计
  - 代码规范和最佳实践
  - 功能模块详细说明（包含音乐系统等）
  - 测试和部署规范

### 专项指南
- **`API_GUIDE.md`** - 本地资源与API配置指南
  - 本地数据结构与加载
  - 可选LLM配置
  - 数据结构和加载机制
  - 监控和故障排除

- **`古代学级称号多语言翻译方案.md`** - 学级升级设定与翻译说明
  - 各学级分数段与中文描述
  - 六语种对照翻译与文化注释
  - 成就弹窗/学级详情的展示规范

- **`TEST_REPORTS.md`** - 测试报告汇总
  - 功能验证测试结果
  - API和数据管理测试
  - 系统集成测试报告
  - 已知问题和改进建议

### 设计资源
- **`design/`** - 设计文档和资源
  - 界面设计截图
  - 数据库设计文档（历史服务器实现，已归档；详见分支 `v1.0-with-backend-server`）
  - 视觉设计资源

## 📖 文档使用指南

### 新开发者入门
1. 首先阅读 `PRD.md` 了解产品定位
2. 查看 `progress.md` 了解项目现状
3. 详读 `DEVELOPMENT.md` 掌握开发规范
4. 根据需要参考专项指南

### 功能开发参考
- 技术实现细节 → `DEVELOPMENT.md`
- 本地资源与LLM配置 → `API_GUIDE.md`
- 测试相关问题 → `TEST_REPORTS.md`

### 项目管理参考
- 进度规划 → `progress.md`
- 需求变更 → `PRD.md`
- 质量评估 → `TEST_REPORTS.md`

## 🔄 文档维护

### 更新频率
- **PRD.md**: 需求变更时更新
- **progress.md**: 每周或重要里程碑时更新
- **DEVELOPMENT.md**: 新功能或规范变更时更新
- **API_GUIDE.md**: API变更或新配置时更新
- **TEST_REPORTS.md**: 测试完成或发现问题时更新

### 维护原则
- 保持文档简洁，避免冗余
- 及时更新，确保信息准确
- 重要变更需要在多个相关文档中同步
- 删除过时信息，避免误导；与历史服务器实现相关内容统一放入“存档说明”章节
